## Part 1: K3s and Vagrant

1. Start the VMs:
   cd p1
   vagrant up

2. SSH into each VM:
   vagrant ssh wilS
   vagrant ssh wilSW

3. On wilS (server), check K3s status:
   sudo k3s kubectl get nodes

4. On wilSW (worker), check node joined:
   sudo k3s kubectl get nodes

5. Both nodes should be Ready and have correct IPs.

---

## Part 2: K3s and Three Simple Applications

1. On the K3s server VM (wilS), apply manifests:
   kubectl apply -f /vagrant/p2/app1-deployment.yaml
   kubectl apply -f /vagrant/p2/app2-deployment.yaml
   kubectl apply -f /vagrant/p2/app3-deployment.yaml
   kubectl apply -f /vagrant/p2/ingress.yaml

2. Check deployments and pods:
   kubectl get deployments
   kubectl get pods
   kubectl get ingress

3. Edit your local /etc/hosts to map app1.com and app2.com to 192.168.56.110.

4. In your browser, visit:
   - http://app1.com (should show app1)
   - http://app2.com (should show app2, 3 replicas)
   - http://192.168.56.110 (should show app3 by default)

---

## Part 3: K3d and Argo CD

1. Run the setup script:
   cd p3
   bash k3d-setup.sh

2. Apply namespaces and Argo CD app:
   kubectl apply -f argocd-namespace.yaml
   kubectl apply -f dev-namespace.yaml
   kubectl apply -f argocd-app.yaml

3. Access Argo CD UI (port-forward if needed):
   kubectl port-forward svc/argocd-server -n argocd 8080:443
   # Then open https://localhost:8080 in your browser.

4. Check that your app is synced and deployed in the dev namespace:
   kubectl get pods -n dev
   kubectl get applications -n argocd
